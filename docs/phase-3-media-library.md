# Phase 3 - 相册 / 媒体库接入

## 背景
Phase 1/2 已分别完成 `localFiles` 与 `localDirectory` 图源能力，用户现在可以从文件选择器或本地目录导入图片并进入现有播放闭环。但在移动设备与 Apple 平台的主流使用路径中，图片通常首先存在系统相册 / 媒体库，而不是用户手动定位文件路径。

为了继续向 ShowcaseApp 的真实图源能力靠拢，Phase 3 将引入 `mediaLibrary` 图源，让用户能够从系统媒体库选择图片资产，并将其保存为可复用、可恢复的图源。当前实现仅承诺在 Android / iOS 提供该入口；macOS 虽属于 Apple 平台使用场景，但本阶段暂未开放媒体库导入入口。

## 目标
- 支持用户在 Android / iOS 从系统相册 / 媒体库选择多张图片
- 生成 `MediaSourceKind.mediaLibrary`
- 保存稳定资产标识与最小展示元数据
- 让媒体库图源进入现有 Sources -> Playback 闭环
- 保证重启恢复、删除回收与失效资产场景不崩溃

## 范围内
- 系统媒体库图片选择入口
- 图片资产多选与 `mediaLibrary` source 生成
- source 持久化与重启恢复
- Playback 对媒体库资产图片的显示
- 删除媒体库图源后的状态回收
- 至少一个支持平台的手工验收

## 不在范围内
- 视频 / Live Photo / GIF 特殊处理
- 智能相册、系统相册分组浏览、搜索与筛选
- 媒体库变更监听、自动刷新与增量同步
- 云同步细节与复杂权限恢复策略
- 离线缓存数据库与原图复制入应用沙盒

## OpenSpec 对应关系
- 主 change：`add-media-library-sources`

## 完成标准
- Sources 页在 Android / iOS 显示“添加媒体库”入口
- 授权后可选择一批图片资产并生成 `mediaLibrary` 图源
- 新图源可进入现有 Sources -> Playback 闭环
- 重启应用后可恢复媒体库图源
- 资产失效时界面不崩溃，至少回退到明确空态或占位态
- 删除当前选中的媒体库图源后状态正确回收

## 当前状态
- 状态：In Progress
- 说明：Phase 3 已建立 docs 与 OpenSpec change，当前开始实现最小闭环版媒体库图源能力，优先覆盖授权、导入、持久化恢复、播放显示与删除回收。

## 手工验收清单

- [ ] 从系统媒体库导入多张图片并生成新图源
- [ ] 导入后自动进入 Playback，且可以看到选中图片
- [ ] 重启应用后媒体库图源仍可恢复

- [ ] 不支持平台（包括当前阶段的 macOS）不显示或不可触发媒体库导入入口
