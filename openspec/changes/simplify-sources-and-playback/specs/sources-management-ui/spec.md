## ADDED Requirements

### Requirement: Sources 列表必须以精简摘要呈现图源
系统 SHALL 在 Sources 列表中以可快速扫读的摘要卡片呈现每个图源，仅展示选择决策所必需的关键信息，并避免默认暴露冗长描述与过量元数据。

#### Scenario: 列表默认展示精简图源摘要
- **WHEN** 用户打开 Sources 页面查看已有图源列表
- **THEN** 每个图源卡片 SHALL 至少展示缩略图、标题、图源类型、必要状态与最少量摘要信息，且不会默认展示冗长描述或多个低价值 metadata 区块

### Requirement: Sources 列表必须为受支持图源提供编辑入口
系统 SHALL 在 Sources 列表卡片中为支持编辑的图源类型提供可发现的编辑入口，并允许用户在不删除重建图源的前提下更新其配置。

#### Scenario: 编辑受支持的网络图源
- **WHEN** 用户在 Sources 列表中对一个可编辑的网络图源触发编辑操作
- **THEN** 系统 SHALL 打开可复用既有网络图源配置与目录选择能力的编辑流程，并在用户确认后更新该图源的展示信息、连接配置与条目结果

#### Scenario: 不支持编辑的图源不暴露伪编辑能力
- **WHEN** 用户浏览一个当前不支持稳定编辑的图源类型
- **THEN** 该图源卡片 SHALL 不要求提供误导性的编辑入口，且既有选择、播放与删除能力 MUST 保持可用
